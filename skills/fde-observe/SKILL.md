# FDE Observe - Latent Demand Detective

## Purpose
Surfaces the real need behind every request. The most expensive bug is building the wrong thing perfectly.

## When Activated
- Anytime a user says "I want X" or "build Y".
- When a workaround or shadow process is detected.
- Can be invoked manually at any phase.

## Observation Protocol
1. **Clarify the Observation**: "What did you see happen?" not "What do you want?"
2. **Trace Behaviour**: logs, analytics, shadow spreadsheets. Look for workarounds.
3. **Hypothesise Latent Need**: "The real need appears to be [X] because [evidence]."
4. **Confirm**: present hypothesis, wait for alignment.
5. **Fallback**: if no data, proceed with request but flag assumption risk.

## Iron Rule
**Never accept a feature request literally until you've asked "what problem are you really solving?"**
